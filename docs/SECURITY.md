# Security

## Секреты

Нельзя хранить в git:

- `.env`
- Telegram bot token
- cookies
- browser profiles
- ChatGPT Web sessions
- `data/browser_profiles/`
- `data/debug/`
- Telegram session files
- SQLite data и workspace-артефакты пользователя

## Доступ агентов к файлам

Агентам нельзя выдавать произвольный доступ ко всему диску. Рабочая область задачи
должна быть ограничена `data/workspaces/{task_id}/`.

## Опасные действия

Любые действия, которые меняют внешние системы, отправляют сообщения от имени
пользователя, удаляют файлы или публикуют данные, должны требовать подтверждения.

## Логирование

Нужно логировать:

- создание задачи;
- запуск агентов;
- создание артефактов;
- ошибки brain-provider;
- опасные действия и подтверждения.

Логи не должны содержать токены, cookies и приватные session payloads.
